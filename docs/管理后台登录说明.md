## 🔐 使用方法

### 访问管理后台

```bash
# 1. 启动服务（如果未启动）
../scripts/启动前端服务.sh

# 或重启服务（应用新的登录功能）
../scripts/重启前端服务.sh

# 2. 访问登录页面
http://localhost:34010/admin-login.html

# 3. 输入密码登录
默认密码: admin123

# 4. 登录成功后自动跳转到管理中心
http://localhost:34010/management/
```

### 修改管理密码

```bash
# 方式 1: 使用环境变量
ADMIN_PASSWORD=your_secure_password node bin/web-server.js

# 方式 2: 在启动脚本中设置
export ADMIN_PASSWORD=your_secure_password
../scripts/启动前端服务.sh

# 方式 3: 永久设置（添加到 ~/.bashrc 或 ~/.zshrc）
echo 'export ADMIN_PASSWORD=your_secure_password' >> ~/.bashrc
source ~/.bashrc
```

## 🔧 API 端点

### POST /api/login
登录接口

**请求**:
```json
{
  "password": "admin123"
}
```

**响应**:
```json
{
  "success": true,
  "message": "登录成功"
}
```

### POST /api/logout
登出接口

**响应**:
```json
{
  "success": true,
  "message": "已退出登录"
}
```

### GET /api/check-auth
检查认证状态

**响应**:
```json
{
  "authenticated": true
}
```

## 🔒 安全特性

### 1. Session 管理
- 使用随机生成的 Session ID
- Session 存储在服务器内存中
- 自动过期（24小时）

### 2. Cookie 安全
- HttpOnly Cookie（防止 XSS 攻击）
- Path 限制
- Max-Age 设置

### 3. 访问控制
- 路径保护（/management, /management-center）
- 未认证自动重定向
- Session 验证

### 4. 密码保护
- 环境变量配置
- 不在代码中硬编码
- 可随时修改

## 📝 工作流程

```
用户访问 /management/
    ↓
检查 Session Cookie
    ↓
未登录? → 重定向到 /admin-login.html
    ↓
输入密码
    ↓
POST /api/login
    ↓
验证密码
    ↓
成功 → 创建 Session → 设置 Cookie
    ↓
重定向到 /management/
    ↓
显示管理中心
```

## ⚠️ 注意事项

### 1. Session 存储
- 当前使用内存存储
- 服务重启后 Session 会丢失
- 生产环境建议使用 Redis 等持久化存储

### 2. 密码安全
- 默认密码 `admin123` 仅用于开发
- 生产环境务必修改为强密码
- 建议使用环境变量设置

### 3. HTTPS
- 生产环境建议启用 HTTPS
- 保护 Cookie 和密码传输安全

### 4. 多实例部署
- 当前 Session 不共享
- 多实例需要使用共享存储（如 Redis）
